## 3. Supabase 테이블 설계

### 3.1 todos 테이블 DDL

```sql
create table todos (
  id         uuid primary key default gen_random_uuid(),
  user_id    uuid references auth.users(id) on delete cascade not null,
  content    text not null,
  is_done    boolean not null default false,
  due_date   date,                          -- KST 기준 날짜 (nullable = 전체 목록용)
  created_at timestamptz not null default now()
);

-- RLS 활성화
alter table todos enable row level security;

-- 본인 데이터만 접근
create policy "users can manage own todos"
  on todos for all
  using (auth.uid() = user_id)
  with check (auth.uid() = user_id);
```

### 3.2 인덱스

```sql
create index todos_user_due on todos(user_id, due_date);
```

## 5. lib/utils/kst.ts 설계

```ts
// 현재 KST 날짜를 'YYYY-MM-DD' 형식으로 반환
export function getTodayKST(): string {
  const now = new Date()
  // UTC+9 보정
  const kst = new Date(now.getTime() + 9 * 60 * 60 * 1000)
  return kst.toISOString().split('T')[0]
}
```

## 6. 컴포넌트 상세 설계

### 6.1 app/todos/page.tsx

```tsx
'use client'

상태:
- todos: Todo[]
- filter: 'today' | 'all'
- loading: boolean

동작:
- 마운트 시 getTodos() 또는 getTodayTodos() 호출
- filter 변경 시 재조회
- addTodo → 목록 prepend (optimistic)
- toggleTodo → 로컬 상태 즉시 반영
- deleteTodo → 로컬 상태에서 제거
```

### 6.2 TodoInput.tsx

```tsx
Props: { onAdd: (content: string) => void }

UI:
- 텍스트 input (placeholder: "할 일을 입력하세요")
- 음성 버튼 (Mic 아이콘, 기존 useSpeechInput 연결)
- 추가 버튼 (Plus 아이콘, 브랜드 컬러 #0F6E56)

동작:
- Enter 또는 추가 버튼 → onAdd(content) 호출
- 음성 인식 결과 → input에 채워짐 → 자동 submit
- AI 인텐트 파싱: parseTodoIntent(text) 통과 시 자동 onAdd
```

### 6.3 FilterTab.tsx (TodoList 내부 포함)

```tsx
Props: { filter: 'today' | 'all', onChange: (f) => void }

UI:
- "오늘" 탭 / "전체" 탭
- 활성 탭: 브랜드 컬러 #0F6E56 하단 선 + 텍스트 강조
```

### 6.4 TodoItem.tsx

```tsx

UI:
- 왼쪽: 완료 체크 아이콘 (Circle → CheckCircle, #0F6E56)
```

## 7. AI 인텐트 파싱 설계

**파일**: `src/lib/utils/parseTodoIntent.ts`

```ts
interface ParseResult {
  isTodoIntent: boolean
  content: string  // 파싱된 할일 내용
}

// 패턴 예시:
// "장보기 할일 추가" → { isTodoIntent: true, content: "장보기" }
// "운동 할일 추가해줘" → { isTodoIntent: true, content: "운동" }
// "할일 추가해줘: 독서" → { isTodoIntent: true, content: "독서" }
// "오늘 날씨 어때?" → { isTodoIntent: false, content: "" }

const patterns = [
  /^(.+)\s*할일\s*추가해?줘?$/,
  /^할일\s*추가해?줘?[:\s]+(.+)$/,
  /^(.+)\s*투두\s*추가해?줘?$/,
]

export function parseTodoIntent(text: string): ParseResult
```

> AI 채팅 컴포넌트(기존)에서 메시지 전송 전 `parseTodoIntent`를 실행하여,
> `isTodoIntent === true`이면 `addTodo(content)`를 호출 후 일반 채팅 흐름 유지.

## 8. 음성 → 투두 흐름

```
사용자가 Mic 버튼 탭
  → useSpeechInput.toggle()
  → 녹음 시작 (lang: 'ko-KR')
  → onResult(transcript) 콜백
  → parseTodoIntent(transcript)
    ├── isTodoIntent === true → addTodo(content) 자동 호출
    └── isTodoIntent === false → input에 텍스트 채움 (수동 submit)
```
