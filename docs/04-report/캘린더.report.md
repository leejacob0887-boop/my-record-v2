## Implementation Details

### Data Model Match (100%)

Supabase `calendar_events` 테이블:
```sql
create table calendar_events (
  id          uuid primary key,
  user_id     uuid references auth.users not null,
  title       text not null,
  date        text not null,           -- 'YYYY-MM-DD' KST
  start_time  text,                    -- 'HH:MM' 24h
  end_time    text,
  description text,
  created_at  timestamptz default now()
);
```

TypeScript 타입과 완벽 일치.

### AI Classification Rules (100%)

`ai-persona.ts` SAVE_ANALYSIS_PROMPT:
- 시간 포함 (오전/오후/시/분) → `calendar_event`
- 마감일만 있음 (오늘까지/내일까지) → `todo`
- JSON 응답 형식 정확히 구현

### Architecture Overview

```
User Input (Chat/Voice)

    ↓
ai-persona.ts (calendar_event 분류)
    ↓
events.ts (CRUD via Supabase)

CalendarDaySheet.tsx (UI 표시)
```

## Lessons Learned

### What Went Well

1. **라이브러리 없는 순수 구현**
   - 외부 라이브러리 의존도 없이 순수 JavaScript 그리드 계산
   - Notia 철학 "AI가 일하기 좋은 구조"에 부합
   - 성능 저하 없음, 번들 크기 증가 미미

2. **AI 양방향 연동**
   - 자연어 파싱(parseEventIntent) 정확도 높음
   - AI save 모드로 자동 저장 시 확인 메시지 친절함
   - 채팅 + 캘린더 두 경로 모두 작동

3. **기존 기능과의 통합**
   - 기존 기록 도트(일기/메모/아이디어) 유지하면서 일정 도트 추가
   - 할일(todos due_date) 자연스럽게 통합
   - BottomTabBar 확장 깔끔함

4. **모바일 최우선 설계**
   - 100dvh 바텀시트로 풍부한 정보 표시
   - 손가락 크기에 맞춘 탭 간격
   - 스크롤 최소화 (한 화면에 정보 집중)

### Areas for Improvement

1. **F-07 직접 음성 입력**
   - 캘린더 페이지 상단에 마이크 버튼 추가하면 UX 더 좋을 것
   - 현재는 채팅 경유만 가능

2. **CalendarGrid.tsx Dead Code**
   - 불필요한 파일 정리 필요
   - 향후 cleanup 단계에서 제거

3. **테스트 커버리지**
   - 자연어 파싱 엣지 케이스 (e.g., "3월 32일", "25시") 테스트 추가 권장
   - 시간 표현 다양성 (e.g., "정오", "자정", "오전 12시") 처리 확장

4. **일정 반복 기능**
   - Plan에 없었지만 향후 "매주 월요일 10시 회의" 같은 반복 일정 유용할 것 같음
   - v1.1에서 고려 가능

### To Apply Next Time

1. **검증 순서 중요**
   - 자연어 파싱 테스트를 먼저 철저히 → 나중에 UI 통합하면 변수 줄어듦

2. **기존 타입과의 일관성**
   - CalendarEvent와 TodoItem 타입 비교 → 통일된 naming convention 유지

3. **AI 규칙 변경 추적**
   - `ai-persona.ts` 변경 시 분류 규칙 주석 상세히 → 나중에 AI가 이해하기 쉬움

4. **도트 표시 우선순위**
   - 기록 도트 + 일정 도트 겹칠 때 어느 걸 앞에 표시할지 일관된 규칙
   - 이번엔 둘 다 표시했는데, 향후 한 도트로 우선순위 표시 고려

## Summary

**캘린더 기능은 Notia의 핵심 가치**인 "당신의 기록을 가장 잘 아는 AI 비서"를 구체화했다.

- **기록 달력** (일기/메모/아이디어 도트): 과거를 돌아보기
- **일정 캘린더** (calendar_events): 미래를 계획하기
- **할일** (todos due_date): 현재를 실행하기

세 가지가 한 캘린더에서 시각화되어, 사용자는 과거·현재·미래를 한눈에 파악할 수 있다.

93% Match Rate로 안정적으로 완료되었으며, 향후 개선 과제(음성 입력, 반복 일정, 카테고리)는 충분히 확장 가능한 아키텍처 위에 구축되어 있다.
