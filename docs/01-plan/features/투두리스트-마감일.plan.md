# 투두리스트 마감일 설정 Plan

## 1. 개요

기존 투두리스트에 마감일(due_date) 설정 기능을 추가. 입력창에서 날짜 picker 또는 자연어("내일까지")로 마감일을 설정하고, 항목 색상으로 마감 상태를 시각화.

## 2. 목표

- 투두 추가 시 날짜 picker로 마감일 선택 가능
- 자연어 입력에서 마감일 자동 파싱 ("내일까지 X", "X월 X일까지 X")
- 마감일 초과 → 빨간색, 마감일 오늘 → 주황색, 정상 → 기본색
- KST 기준 날짜 비교
- 기존 투두 컴포넌트 최소 수정

## 3. 요구사항

### 3.1 기능 목록

| ID | 기능 | 우선순위 |
|----|------|---------|
| F-01 | 날짜 picker로 마감일 선택 | P0 |
| F-02 | 자연어 입력 → 마감일 자동 파싱 | P0 |
| F-03 | 마감일 초과 시 빨간색 표시 | P0 |
| F-04 | 마감일 오늘 시 주황색 표시 | P0 |
| F-05 | 마감일 표시 (TodoItem에 날짜 뱃지) | P0 |
| F-06 | 마감일 없는 투두는 기본 스타일 유지 | P0 |

### 3.2 자연어 파싱 패턴

| 입력 예시 | 파싱 결과 |
|----------|----------|
| "내일까지 보고서 제출" | due_date = 내일, content = "보고서 제출" |
| "오늘까지 세금 납부" | due_date = 오늘, content = "세금 납부" |
| "3월 25일까지 발표 준비" | due_date = 03-25, content = "발표 준비" |
| "이번 주 금요일까지 코드 리뷰" | due_date = 이번 주 금요일, content = "코드 리뷰" |
| "장보기" | due_date = null (파싱 없음), content = "장보기" |

### 3.3 색상 규칙

| 상태 | 조건 | 색상 |
|------|------|------|
| 마감 초과 | due_date < today (KST) | `text-red-500`, 뱃지 `bg-red-100` |
| 마감 오늘 | due_date === today (KST) | `text-orange-500`, 뱃지 `bg-orange-100` |
| 마감 예정 | due_date > today (KST) | 기본색, 뱃지 `bg-gray-100` |
| 마감일 없음 | due_date === null | 기본 스타일 |

## 4. 기술 결정

| 항목 | 결정 | 이유 |
|------|------|------|
| 날짜 입력 | `<input type="date">` | 별도 라이브러리 없이 모바일 네이티브 picker |
| 자연어 파싱 | 클라이언트 정규식 | 간단한 패턴만 처리, API 불필요 |
| 날짜 비교 | KST getTodayKST() 재사용 | 기존 유틸 활용 |
| DB 변경 없음 | todos.due_date 컬럼 이미 존재 | 스키마 수정 불필요 |
