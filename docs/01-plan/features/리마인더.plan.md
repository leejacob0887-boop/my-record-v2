# 리마인더 Plan

## 1. 개요

Notia AI 채팅/음성 입력에서 시간 관련 표현("내일 오전 9시 약 먹어야 해")을 자동 감지하고,
사용자 동의("응") 시 FCM 푸시 알림을 등록하는 기능.
캘린더 일정에도 알림 설정 가능. KST 기준 처리. 기존 코드 무변경 원칙.

## 2. 목표

- AI 채팅/음성에서 시간 표현 감지 → 자동으로 "알림 설정해드릴까요?" 제안
- 사용자 확인("응", "네", "설정해줘") → FCM 토픽/토큰으로 예약 알림 등록
- 캘린더 일정 추가/편집 시 알림 시간 설정 가능
- 브라우저 Notification 권한 요청 (최초 1회)
- PWA 서비스 워커를 통한 백그라운드 푸시 수신
- KST 기준 시각 계산

## 3. 요구사항

### 3.1 데이터 (Supabase)

**reminders 테이블**

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK, auto |
| user_id | uuid | FK → auth.users |
| title | text | 알림 내용 |
| remind_at | timestamptz | KST 기준 알림 시각 (UTC 저장) |
| fcm_token | text | 브라우저 FCM 토큰 |
| source | text | 'chat' \| 'calendar' |
| source_id | uuid | 연결된 calendar_events.id (nullable) |
| is_sent | boolean | 발송 완료 여부 (default: false) |
| created_at | timestamptz | 생성 시각 |

### 3.2 기능 목록

| ID | 기능 | 우선순위 |
|----|------|---------|
| F-01 | 채팅 입력에서 시간 표현 감지 | P0 |
| F-02 | "알림 설정해드릴까요?" 제안 메시지 표시 | P0 |
| F-03 | 사용자 동의 처리 ("응"/"네"/"설정해줘") | P0 |
| F-04 | FCM 토큰 취득 및 저장 | P0 |
| F-05 | Supabase reminders 테이블에 알림 등록 | P0 |
| F-06 | 브라우저 Notification 권한 요청 | P0 |
| F-07 | PWA 서비스 워커 푸시 수신 핸들러 | P1 |
| F-08 | 캘린더 일정 알림 설정 (EventForm 확장) | P1 |
| F-09 | 알림 발송 트리거 (Supabase Edge Function 또는 Vercel Cron) | P1 |

### 3.3 시간 표현 감지 규칙

기존 `parseEventIntent.ts`와 유사한 `parseReminderIntent.ts` 신규 파일로 분리.

| 패턴 | 예시 |
|------|------|
| 내일/오늘/모레 + 시각 | "내일 오전 9시 약 먹어야 해" |
| 요일 + 시각 | "다음주 월요일 2시까지" |
| N월 N일 + 시각 | "3월 25일 오후 3시" |
| "~해야 해", "~잊지 마", "~알려줘" | 알림 의도 강화 키워드 |

### 3.4 사용자 동의 플로우

```
사용자 입력 → parseReminderIntent 감지
  → 채팅창에 제안 버튼 표시:
    [알림 설정] [괜찮아요]
  → "응"/"네" 입력 또는 [알림 설정] 클릭
  → 브라우저 알림 권한 요청 (미허용 시 안내)
  → FCM 토큰 취득
  → reminders 테이블 INSERT
  → "✅ [내일 오전 9시]에 알림을 설정했어요!" 확인 메시지
```

### 3.5 FCM 설정

| 항목 | 내용 |
|------|------|
| SDK | firebase/app, firebase/messaging |
| 설정 파일 | src/lib/firebase.ts (신규) |
| 서비스 워커 | public/firebase-messaging-sw.js (신규) |
| 환경 변수 | NEXT_PUBLIC_FIREBASE_* (6개) |
| 알림 발송 | Supabase Edge Function `send-reminder` 또는 Vercel Cron |

### 3.6 캘린더 연동

- `EventForm.tsx`에 "알림 시간" 필드 추가 (remind_at: 일정 N분 전)
- 일정 저장 시 reminders 테이블에도 자동 INSERT
- 캘린더 일정 삭제 시 연관 reminder도 삭제

## 4. 기술 스택

| 기술 | 용도 |
|------|------|
| Firebase Cloud Messaging | 푸시 알림 토큰 관리 및 발송 |
| Web Push API | PWA 백그라운드 알림 수신 |
| Notification API | 브라우저 권한 요청 |
| Supabase | reminders 테이블 저장 |
| Supabase Edge Function | 예약 시각 도달 시 FCM 발송 트리거 |
| firebase/app, firebase/messaging | Next.js 클라이언트 FCM SDK |

## 5. 비기능 요구사항

- 기존 코드(chat/page.tsx, calendar/page.tsx 등) 직접 수정 없이 hook/provider 방식으로 연결
- parseReminderIntent는 parseEventIntent와 독립적인 별도 파일
- 알림 권한 미허용 시 UI 안내 메시지 표시 (에러 throw 금지)
- KST 기준 시각 → UTC 변환 후 저장 (`remind_at - 9h`)
