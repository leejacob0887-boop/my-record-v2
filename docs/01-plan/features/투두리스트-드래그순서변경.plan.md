# 투두리스트 드래그 순서 변경 Plan

## 1. 개요

투두 목록에서 아이템을 드래그해 순서를 자유롭게 변경하는 기능.
모바일(터치) + PC(마우스) 모두 지원.

## 2. 요구사항

| ID | 기능 | 우선순위 |
|----|------|---------|
| F-01 | 드래그앤드롭으로 투두 순서 변경 | P0 |
| F-02 | 모바일 터치 이벤트 지원 | P0 |
| F-03 | 변경된 순서를 Supabase에 저장 (새로고침 유지) | P0 |
| F-04 | 드래그 중 시각적 피드백 (드래그 중인 아이템 강조) | P1 |
| F-05 | 기존 우선순위 자동정렬과 공존 (수동 정렬 시 우선순위 정렬 비활성화) | P1 |

## 3. 기술 결정

| 항목 | 결정 | 이유 |
|------|------|------|
| DnD 라이브러리 | `@dnd-kit/core` + `@dnd-kit/sortable` | 모바일 터치 완벽 지원, 가볍고 접근성 우수 |
| 순서 저장 | Supabase `todos` 테이블에 `sort_order` 컬럼 추가 | 새로고침 후에도 순서 유지 |
| 정렬 전략 | sort_order 기준 정렬 (우선순위 정렬은 초기 추가 시에만 적용) | 수동 순서와 자동 우선순위 공존 |
| 터치 센서 | `TouchSensor` (delay: 250ms, tolerance: 5px) | 모바일 스크롤과 드래그 구분 |

## 4. Supabase SQL

```sql
alter table todos add column sort_order integer;

-- 기존 투두에 순서 부여 (created_at 기준)
update todos
set sort_order = sub.rn
from (
  select id, row_number() over (partition by user_id order by created_at desc) as rn
  from todos
) sub
where todos.id = sub.id;
```

## 6. 구현 상세

### 6.1 패키지 설치
```bash
npm install @dnd-kit/core @dnd-kit/sortable @dnd-kit/utilities
```

### 6.2 순서 업데이트 함수 (todos.ts 추가)
```ts
export async function updateTodoOrder(id: string, sort_order: number): Promise<void> {
  const { error } = await supabase
    .from('todos')
    .update({ sort_order })
    .eq('id', id)
  if (error) throw error
}
```

### 6.3 TodoList — DnD 래퍼
- `DndContext` (sensors: MouseSensor + TouchSensor)
- `SortableContext` (items: todo ids, strategy: verticalListSortingStrategy)
- `onDragEnd`: `arrayMove`로 순서 변경 후 각 아이템 `sort_order` 일괄 업데이트

### 6.4 TodoItem — Sortable 적용
- `useSortable(id)` 훅
- `transform`, `transition` 스타일 적용
- GripVertical 아이콘(lucide-react)을 드래그 핸들로 사용
