# 투두리스트 Plan

## 1. 개요

Notia 앱 안에서 할 일을 추가·완료·삭제하고, AI 채팅 및 음성 입력으로 자동 저장되는 투두리스트 기능.

## 2. 목표

- Supabase `todos` 테이블 기반 CRUD
- 오늘 할일 / 전체 할일 필터링
- AI 채팅에서 자연어로 투두 추가 ("할일 추가해줘")
- 음성 입력으로 투두 추가 ("장보기 할일 추가")
- KST 기준 날짜 처리
- 기존 기록(Record) 기능에 영향 없이 독립 추가

## 3. 요구사항

### 3.1 데이터 (Supabase)

**todos 테이블**

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK, auto |
| user_id | uuid | FK → auth.users |
| content | text | 할일 내용 |
| is_done | boolean | 완료 여부 (default: false) |
| due_date | date | 대상 날짜 (KST 기준) |
| created_at | timestamptz | 생성 시각 |

### 3.2 기능 목록

| ID | 기능 | 우선순위 |
|----|------|---------|
| F-01 | 투두 추가 (텍스트 입력) | P0 |
| F-02 | 투두 완료 체크/해제 | P0 |
| F-03 | 투두 삭제 | P0 |
| F-04 | 오늘 할일 필터 (KST 기준) | P0 |
| F-05 | 전체 할일 필터 | P0 |
| F-06 | AI 채팅에서 할일 자동 추가 | P1 |
| F-07 | 음성 입력으로 할일 추가 | P1 |

### 3.3 UI/UX 요구사항

- 브랜드 컬러 #0F6E56 사용 (완료 체크, 버튼 등)
- 아이콘: lucide-react (`CheckCircle`, `Circle`, `Trash2`, `Plus`, `Mic`)
- 모바일 최우선 레이아웃 (100dvh)
- 기존 홈/기록 화면과 독립된 탭 또는 섹션

## 4. 기술 결정

| 항목 | 결정 | 이유 |
|------|------|------|
| DB | Supabase (기존) | 인증·RLS 활용 |
| 날짜 | KST (UTC+9) 보정 | 한국 사용자 기준 |
| AI 파싱 | 클라이언트 정규식 | 간단한 패턴 ("X 할일 추가") |
| 음성 | Web Speech API | 별도 라이브러리 없이 처리 |
| 상태 관리 | React useState + Supabase realtime | 기존 패턴 유지 |

## 5. 구현 범위 (In/Out)

### In Scope
- todos CRUD (Supabase)
- 오늘/전체 필터
- AI 채팅 인텐트 감지 → 투두 추가
- 음성 → 텍스트 → 투두 추가
- KST 날짜 처리

### Out of Scope
- 반복 투두 (추후)
- 알림/푸시 (추후)
- 투두 순서 정렬 drag-and-drop (추후)
- 공유 투두 (추후)

## 7. 주요 제약

- 기존 `records`, `home` 관련 파일 수정 금지
