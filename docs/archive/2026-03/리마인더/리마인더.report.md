## 2. Executive Summary

### 2.1 핵심 성과

이 기능은 "말하면 알아서 알림까지"라는 Notia의 핵심 철학을 구현한다. 사용자가 AI 채팅이나 음성으로 시간 표현("내일 오전 9시 약 먹어야 해")을 입력하면, 시스템이 자동으로 감지해 "알림 설정해드릴까요?"를 제안하고, 사용자가 "응" 한 마디로 확인하면 FCM 푸시 알림이 등록된다. 캘린더에서도 직접 알림을 설정할 수 있으며, 매분 Vercel Cron이 실행되어 지정된 시각에 자동으로 푸시 알림을 발송한다.

### 2.2 Value Delivered (4-Perspective)

| 관점 | 내용 |
|------|------|
| **Problem** | 기록했어도 실행 시각을 놓쳐 잊어버리는 문제 — "내일 약 먹어야 해"를 기록해도 실제 시간에 알림 없음 |
| **Solution** | AI가 자연어에서 시간 표현을 자동 감지 + "응"으로 즉시 알림 등록 + Vercel Cron 매분 발송 (기존 코드 무변경) |
| **Function/UX Effect** | 채팅 중 "내일 오전 9시 약" 입력 → 제안 UI + 캘린더에서도 [알림 추가] FAB → 브라우저 알림권한 → FCM 토큰 → 예약 알림 완료 |
| **Core Value** | 기록에서 실행까지 끊김 없는 흐름으로 사용자 행동률 극대화. PWA 백그라운드 푸시 지원으로 항상 깨어있는 AI 비서 실현. |

## 6. 주요 기술 결정 사항

### 6.1 localStorage 폴링 vs 다른 방식

**선택**: localStorage 폴링 (800ms 간격)

**근거**:
- 프론트-백엔드 분리 유지
- 기존 chat/page.tsx 수정 없음
- 간단한 구현, 낮은 유지보수 비용

**대안 검토**:
- WebSocket: 복잡도 높음, 백엔드 추가 구현 필요
- API 폴링: network overhead 큼
- 메시지 후킹: page.tsx 수정 필요 (회피)

### 6.2 Vercel Cron vs Supabase Edge Function

**선택**: Vercel Cron (`/api/cron/send-reminders`)

**근거**:
- Next.js 프로젝트에 기본 포함
- 배포 간편 (vercel.json 추가만)
- 매분 실행 간격 충분
- Edge Function 보다 디버깅 용이

### 6.3 autoConfirm 상태 관리

**선택**: React useState로 pending/autoConfirm 분리

**근거**:
- pending: 제안 UI 표시 여부
- autoConfirm: 사용자 텍스트 입력 동의 감지 시 true로 설정
- ReminderPrompt가 autoConfirm 감지 시 자동 확인 버튼 실행

## 7. 남은 구성 및 설정

### 7.1 Vercel 환경변수 (필수)

프로젝트 배포 시 Vercel 대시보드에서 설정 필요:

```
CRON_SECRET=<임의 문자열>           # Cron 요청 검증
SUPABASE_SERVICE_ROLE_KEY=<key>     # Server-side DB 접근
FIREBASE_PROJECT_ID=<id>
FIREBASE_PRIVATE_KEY=<key>
FIREBASE_CLIENT_EMAIL=<email>
```

### 7.2 Supabase reminders 테이블

SQL 마이그레이션:

```sql
CREATE TABLE public.reminders (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid NOT NULL REFERENCES auth.users(id),
  title text NOT NULL,
  remind_at timestamptz NOT NULL,
  fcm_token text,
  source text DEFAULT 'chat', -- 'chat' | 'calendar'
  source_id uuid,
  is_sent boolean DEFAULT false,
  created_at timestamptz DEFAULT now(),

  FOREIGN KEY (source_id) REFERENCES calendar_events(id) ON DELETE SET NULL
);

CREATE INDEX idx_reminders_user_id ON reminders(user_id);
CREATE INDEX idx_reminders_remind_at ON reminders(remind_at, is_sent);
```

### 7.3 Firebase 프로젝트 설정

1. Firebase Console: Web 앱 등록
2. VAPID 공개키 생성 (Cloud Messaging 탭)
3. 환경변수 저장: `NEXT_PUBLIC_FIREBASE_*` (6개)
