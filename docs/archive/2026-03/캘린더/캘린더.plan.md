# 캘린더 Plan

## 1. 개요

모바일 최적화 월간 캘린더. 라이브러리 없이 직접 구현.
일정(calendar_events) + 투두(todos due_date)를 날짜별로 통합 표시.
AI 채팅·음성 입력으로 자연어 일정 추가 지원.

## 3. 데이터 모델

### Supabase 테이블: calendar_events

```sql
create table calendar_events (
  id          uuid primary key default gen_random_uuid(),
  user_id     uuid references auth.users not null,
  title       text not null,
  date        text not null,           -- 'YYYY-MM-DD' KST
  start_time  text,                    -- 'HH:MM' 24h, nullable
  end_time    text,                    -- 'HH:MM' 24h, nullable
  description text,
  created_at  timestamptz default now()
);

alter table calendar_events enable row level security;
create policy "본인 일정만" on calendar_events
  for all using (auth.uid() = user_id);
```

### TypeScript 타입

```ts
export interface CalendarEvent {
  id: string
  user_id: string
  title: string
  date: string           // 'YYYY-MM-DD'
  start_time: string | null  // 'HH:MM'
  end_time: string | null
  description: string | null
  created_at: string
}
```

## 5. 자연어 파싱 (parseEventIntent.ts 신규)

지원 패턴:
- `"내일 오후 3시 병원"` → date: 내일, start_time: 15:00, title: 병원
- `"3월 25일 오전 10시 회의"` → date: 3월 25일, start_time: 10:00, title: 회의
- `"오늘 저녁 7시 밥약속"` → date: 오늘, start_time: 19:00, title: 밥약속
- `"다음주 월요일 2시 미팅"` → date: 다음주 월요일, start_time: 14:00, title: 미팅

시간 표현 파싱:
- 오전 N시 → N:00 (N < 12)
- 오후 N시 → (N+12):00 (N < 12)
- N시 M분 → N:M
- 저녁 → pm, 아침 → am

## 7. 캘린더 그리드 구현 방식

```
CalendarGrid 계산 로직:
1. 해당 월 1일 요일 구하기 (0=일 ~ 6=토)
2. 총 날짜 수: daysInMonth(year, month)
3. 6행 × 7열 = 42칸 그리드
4. 빈 칸(이전 달 말일, 다음 달 초) 회색 처리
5. 각 날짜 셀에: 숫자 + 도트(일정 있으면) 표시
```
